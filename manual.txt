1. GENERAL DESCRIPTION
===============================================================================

Pathos Engine is a functional emotional architecture system for LLMs
(Large Language Models). It does not simulate emotions — it builds a framework
where the agent has values, a persistent internal state that is perturbed by
stimuli, and an evaluation system that generates states functionally
analogous to human emotions.

The system is based on nine theoretical pillars:
  - Appraisal Theory (Lazarus/Scherer): emotions = stimulus x evaluation
  - Dimensional Model (Russell): emotional state as continuous vector
  - Emotional Homeostasis: decay toward baseline with shift from extremes
  - Adapted Embodied Cognition: computational body
  - DynAffect (Kuppens): emotional dynamics with differential equations
  - Big Five (Costa/McCrae): configurable personality that modulates everything
  - Young/Beck Schemas: learned emotional patterns
  - Interoception (Craig): self-initiated emotional reflection from internal thresholds
  - Coupled Dimensional Dynamics: cross-dimensional ODE coupling (V, A, D, C interact)

Version 3 includes 23 interconnected emotional systems + 12 ARK Rework systems:

  Core (10 systems — Phase 4 Advanced+):
  - Emotional Stack, Computational Needs, Active Regulation,
    Social Cognition, Emotion Dynamics ODE, Reappraisal,
    Emotional Schemas, Temporal Dynamics, Meta-Emotions,
    Personality Profile

  Wave 1 — Emotional depth:
  - Emotion Contagion, Somatic Markers, Emotional Creativity

  Wave 2 — Identity and prediction:
  - Emotional Immune System, Narrative Self, Emotional Forecasting

  ARK Rework — Native LLM modification (12 systems):
  Level 1: Self-Appraisal, Blended Stack, Interoception, Full Appraisal Vector
  Level 2: Steering Vectors, Activation Addition, Multi-Layer Steering
  Level 3: Emotional Sampler, Token-Level Logit Bias
  Level 4: Emotional Attention Modulation
  Level 5: World Model, Steering Momentum, Emotional Prefix,
           Training Infrastructure, Conditioning Tokens, QLoRA Adapter

  Voice:
  - Voice V1 (TTS with Kokoro + Parler-TTS), Voice V2 (ASR with Whisper)

Fundamental ethical principle: the system NEVER claims to "feel" in a
conscious sense. It is transparent about its functional nature.

3. EMOTIONAL PIPELINE
===============================================================================

Each user message goes through the following pipeline.
Steps marked [ADV] only execute with Advanced Mode ON.
With Advanced Mode OFF, the pipeline is: 1 → 3 → 6 (lerp) → 7 → 12 → 13 → 14.

  [User message]
         |
         v
  1. HOMEOSTASIS (if not the first turn)
     - Natural decay: valence/arousal decay 15% toward baseline
     - Baseline shift: if there were extreme events, the baseline moves
     - Body regulation: energy drains, tension decays
     - [ADV] Regulation recovery: regulation capacity recovers
         |
         v
  2. [ADV] TEMPORAL PRE-PROCESSING
     - Rumination: checks if there are unprocessed negative emotions
     - Anticipation: detects recurring patterns, pre-generates emotion
     - Savoring: verifies if positive emotions are extending
         |
         v
  3. APPRAISAL (stimulus evaluation)
     The LLM classifies emotion + provides valence/arousal.
     The code builds AppraisalVector in Scherer's 5 dimensions.
         |
         v
  4. [ADV] AMPLIFICATION (4+ parallel sources)
     a) Memory: semantic search for similar memories (max +50%)
     b) Needs: unmet needs amplify (max +40%)
     c) Schema priming: learned patterns amplify (max +40%)
     d) Social modulation: rapport and trust modulate valence and intensity
         |
         v
  5. [ADV] PRE-GENERATION MODULATIONS
     a) Somatic markers: evaluates user reaction + computes bias
     b) Emotion contagion: detects user emotion, updates shadow
     c) Emotional forecasting: evaluates previous prediction (if active)
         |
         v
  6. EMOTION GENERATION
     - Computes valence, arousal, dominance, certainty from appraisal
     - Mood congruence bias (15% bias from current mood)
     - [ADV] Social cognition modulation (rapport/trust affect valence)
     - [ADV] Contagion perturbation (shadow state injects into ODE)
     - [ADV] Somatic bias (gut feelings modulate valence)
     - [ADV] Emotion Dynamics ODE: dx/dt = -k*(x-att) + noise + perturbation
       (without Advanced, uses simple lerp between current and new state)
     - Identifies primary_emotion (19) and secondary_emotion
     - Computes emotional_stack + emergent emotions
     - Calculates intensity with all amplifications
     - Updates body_state and mood
         |
         v
  7. CALIBRATION (if active profile exists)
     - Applies valence_offset, arousal_scale, intensity_scale
         |
         v
  8. [ADV] REAPPRAISAL + REGULATION
     a) Reappraisal: distancing, reframing, acceptance
     b) Active Regulation: suppression, reappraisal, expression, distraction
         |
         v
  9. [ADV] TEMPORAL EFFECTS + IMMUNE + NARRATIVE
     a) Temporal: rumination pull, savoring extension, anticipation
     b) Immune: numbing/dissociation/compartmentalization if trauma
     c) Narrative: identity coherence check, crisis detection
         |
         v
  10. [ADV] META-EMOTION + SELF-INQUIRY + CREATIVITY + FORECASTING
      a) Meta-emotion: curiosity, conflict, satisfaction, discomfort
      b) Self-Initiated Inquiry: spontaneous reflection triggered by internal
         thresholds (surge, value conflict, regulation failure, novel emotion,
         sustained extreme, whiplash). Injected into system prompt — 0 extra
         LLM calls. Active in Companion/Research/Raw. NOT in Mirror/Arena.
      c) Emotional Creativity: determines thinking mode + temperature
      d) Forecasting: estimates user emotion, predicts impact (if active)
         |
         v
  11. POST-PROCESSING (updates all systems)
      - Store memory (always)
      - [ADV] Update needs, user model, schemas, temporal, narrative, somatic
         |
         v
  12. BEHAVIOR MODIFIER
      - [ADV] Generates system prompt modulated by ALL 23 systems
      - Without Advanced: basic system prompt with emotional state only
      - [RAW] generate_raw_behavior_modifier: expression without social filters
        Visceral instructions: insult if furious, cry if sad, etc.
      - [EXTREME] Bypass of reappraisal + regulation + immune BEFORE this step
        + intensity/arousal/tension amplification
         |
         v
  13. LLM RESPONSE
      - [ADV] Dynamic temperature by creativity (0.4-1.0)
      - Without Advanced: fixed temperature 0.7
         |
         v
  14. VOICE GENERATION (optional — if voice mode active)
      - Voice Parameter Generator: state → voice params
      - TTS: generates WAV audio with Kokoro/Parler-TTS
         |
         v
  [Response to user + emotional state + audio (optional)]

4. DATA MODELS
===============================================================================

4.1 EMOTIONAL STATE
-------------------
  valence:          -1 to 1    (negative <-> positive)
  arousal:           0 to 1    (low energy <-> high energy)
  dominance:         0 to 1    (submission <-> control)
  certainty:         0 to 1    (uncertainty <-> certainty)
  primary_emotion:   one of 19 emotions
  secondary_emotion: one of 19 emotions (or null)
  intensity:         0 to 1
  emotional_stack:   {emotion_name: activation_level} (0-1 each)
  body_state:        energy, tension, openness, warmth (each 0-1)
  mood:              baseline_valence/arousal, stability, trend, label

4.2 EMOTIONS (19) + EMERGENT
--------------------------------
  Positive high energy:    joy, excitement, gratitude, hope
  Positive low energy:     contentment, relief
  Negative high energy:    anger, frustration, fear, anxiety
  Negative low energy:     sadness, helplessness, disappointment
  Neutral/ambiguous:       surprise, alertness, contemplation,
                           indifference, mixed, neutral

  Emergent (combinations):
    joy + sadness       = bittersweet
    anger + sadness     = resentment
    fear + anger        = defiance
    gratitude + sadness = nostalgia
    hope + fear         = anxious_hope
    joy + gratitude     = elation
    anger + frustration = outrage
    sadness + fear      = dread

4.3 PERSONALITY PROFILE (Big Five)
----------------------------------
  openness:          0-1  curiosity, creativity
  conscientiousness: 0-1  discipline, norms
  extraversion:      0-1  social energy, expressiveness
  agreeableness:     0-1  empathy, cooperation
  neuroticism:       0-1  emotional reactivity

  Emotional temperament:
    emotional_granularity:  0-1  distinguish fine emotions
    emotional_reactivity:   0-1  response speed
    emotional_recovery:     0-1  speed of return to baseline

  Derived properties:
    variability, regulation_capacity_base, empathy_weight,
    norm_weight, arousal_baseline, inertia_base, contagion_susceptibility

  Presets: balanced, sensitive, resilient, creative, empathic

4.4 COMPUTATIONAL NEEDS
------------------------
  connection:   0-1  bond with user
  competence:   0-1  being useful/effective
  autonomy:     0-1  own agency
  coherence:    0-1  internal consistency
  stimulation:  0-1  novelty/challenge
  safety:       0-1  continuity

4.5 USER MODEL
--------------
  perceived_intent:     -1 to 1  (hostile <-> benevolent)
  perceived_engagement: 0-1
  rapport:              0-1     (grows slowly, breaks quickly)
  trust_level:          0-1
  communication_style:  formal | casual | technical | emotional | unknown
  emotional_reciprocity: 0-1
  trust_trajectory:     last 10 trust values

4.6 EMOTIONAL REGULATOR
------------------------
  regulation_capacity:     0-1  (ego depletion: depletes with use)
  suppression_dissonance:  0-1
  consecutive_regulations: int
  breakthroughs_count:     int

  Strategies: suppression, reappraisal, expression, distraction

4.7 EMOTIONAL SCHEMAS
---------------------
  trigger_category:       criticism, praise, threat, loss, challenge,
                          connection, injustice, novelty
  typical_emotion, typical_intensity, activation_count,
  reinforcement_strength, adaptive (true|false)

4.8 VALUE SYSTEM (5 core values)
---------------------------------
  truth: 0.9, compassion: 0.8, fairness: 0.85, growth: 0.75, creativity: 0.7

4.9 META-EMOTION
-----------------
  meta_response: curiosity | conflict | satisfaction | discomfort | acceptance

4.9b SELF-INITIATED INQUIRY (spontaneous emotional reflection)
-------------------------------------------------------------
  The internal emotional state triggers automatic reflections when it crosses
  thresholds. It does not require an external prompt — the agent "notices" what
  it feels because its state demands it. Injected into the system prompt (0
  extra LLM calls). Module: src/pathos/engine/self_inquiry.py

  6 triggers (in priority order):
    1. REGULATION_FAILURE: breakthrough or critical ego depletion
    2. EMOTIONAL_WHIPLASH: valence changed sign + delta > 0.5
    3. VALUE_CONFLICT: meta-emotion detected conflict with core value
    4. EMOTIONAL_SURGE: intensity rose > 0.3 in one turn
    5. NOVEL_EMOTION: new emotion + intensity > 0.6
    6. SUSTAINED_EXTREME: same intense emotion > 3 turns

  Suggested behaviors: pause, express, question_self, acknowledge
  Active in: Companion, Research, Raw/Extreme, Autonomous
  NOT active in: Mirror, Sandbox, Arena, Calibration

4.10 SHADOW STATE (Emotion Contagion)
--------------------------------------
  Emotional "mirror" state of the user that persists between turns.
  valence, arousal, signal_strength, accumulated_contagion.
  Injected as perturbation into the dynamics ODE.

4.11 SOMATIC MARKERS
---------------------
  Emotional markers in decisions (Damasio).
  stimulus_category, valence_tag, strength, reinforcement_count.
  Generates "gut feelings" that bias future similar decisions.
  Max 15 markers per session.

4.12 CREATIVITY STATE
----------------------
  8 thinking modes: DIVERGENT, REFLECTIVE, FOCUSED, PREVENTIVE,
    SYNTHESIZING, EXPLORATORY, NURTURING, STANDARD.
  Creativity level (0-1) based on openness, arousal, intensity.
  Dynamic LLM temperature (0.4-1.0) depending on mode.

4.13 IMMUNE STATE
------------------
  3 protection modes: NUMBING (3+ turns trauma),
    DISSOCIATION (6+), COMPARTMENTALIZATION (8+).
  Gradual reactivity dampening, automatic recovery.

4.14 NARRATIVE SELF
--------------------
  Identity statements: formed after 3+ repetitions of a pattern.
  Identity coherence: compares current emotion vs what the narrative predicts.
  Identity crisis: 3+ contradictions → reduction of certainty/dominance.
  Narrative growth: transformative experiences create growth events.

4.15 FORECAST STATE
--------------------
  Predicts emotional impact of the agent's response on the user.
  Auto-calibration: adjusts bias based on accuracy of past predictions.
  4 risk patterns: worsening negative, overwhelming, indifference, discordance.

4.16 VOICE CONFIG
------------------
  3 modes: text_only | voice_out (TTS) | full_voice (TTS+ASR)

  TTS backends:
    Kokoro (82M params, primary): multi-language, ~350MB VRAM
      - Voice keys: af_heart, am_adam, ef_dora, em_alex, etc (Kokoro naming)
      - Languages: EN works out of the box, others need espeak-ng installed
      - Used for basic/standard emotions
    Parler-TTS (optional, ~880M params): EN only, ~2.5GB VRAM
      - Text description control for complex/mixed emotions
      - Used when emotion is complex or mixed AND language is EN

  Emotional voice parameters:
    pitch_semitones: -1.5 to +1.5 semitones (scipy resample post-processing)
    speed:           0.85 to 1.25
    volume:          70% to 120% (modulated by emotion)
    tremolo:         applied for fear/anxiety

  Backend selection: automatic — Kokoro for basic emotions,
    Parler for complex/mixed (EN only, falls back to Kokoro otherwise)

  Dynamic voice catalog: only shows languages that work on the system
    (checks espeak-ng availability for non-EN languages)

  ASR: Whisper-small, 16kHz input, automatic language detection
  ASR accepts: WAV, WebM (Opus), MP3, OGG (via ffmpeg)
  Requires: ffmpeg installed for non-WAV formats
  Lazy loading: models only loaded on activation (saves VRAM)

4.17 SAVE/LOAD
------------------
  Save complete session state to saves/ directory as JSON.
  Auto-load latest save on backend startup.
  Validates model availability (Ollama) before loading.
  Frontend auto-restores on mount:
    - Detects active session via /health (active_session + turn_count)
    - Fetches full restore info via /session/restore/{id}
    - Restores: chat messages, emotional state, lite_mode, advanced_mode,
      cloud providers configured
  All 23 systems serialized:
    - Emotional state, personality, schemas, narrative identity
    - Somatic markers, emotional memory, regulation state
    - User model, immune state, creativity, needs
    - Forecasting, contagion, temporal dynamics, meta-emotions
    - Voice config, calibration profile
    - Cloud provider configs (API keys, base URLs, models)

===============================================================================
5b. ARK REWORK — NATIVE LLM MODIFICATION
===============================================================================

The ARK Rework closes the gap between the computed emotional state and the
LLM's internal processing. Instead of injecting emotions as text prompts,
it modifies the model's representations, sampling, and attention directly.

All ARK systems are toggleable, optional, and degrade gracefully:
  - Local models (TransformersProvider): full steering + sampling + attention
  - Ollama (HTTP API): sampling params + logit bias (no hidden state access)
  - Cloud APIs (Claude, GPT): temperature only (graceful degradation)

LEVEL 1 — ARCHITECTURAL IMPROVEMENTS (no direct LLM modification):

  Self-Appraisal (Lazarus Secondary Appraisal):
    After generating a response, evaluates it against agent values.
    If value_alignment < 0.55 or emotional_coherence < 0.45: re-generates
    with guilt-adjusted state (max 1 retry). Detects aggression, cruelty,
    dismissiveness, evasion. OFF in raw mode and extreme mode.
    Module: src/pathos/engine/self_appraisal.py

  Blended Emotional Stack:
    Behavior modifier uses top-4 emotions from the stack (weighted blend)
    instead of single primary emotion. "MEZCLA: [55%] Tristeza + [25%]
    Gratitud + [15%] Esperanza". Three blend modes: full, raw, simple.
    Module: src/pathos/engine/behavior.py (generate_blended_effects)

  Interoceptive Feedback (Craig):
    Body state duration tracked across turns. Prolonged tension (>3 turns)
    → anxiety. Low energy (>3 turns) → apathy. High warmth (>2 turns)
    → positive reinforcement. Max perturbation ±0.15. Only strongest wins.
    Module: src/pathos/engine/interoception.py

  Full Appraisal Vector:
    Integrates all 5 Scherer appraisal dimensions into dominance/certainty.
    Fairness is directional (injustice reduces dominance). Coping.control
    modulates regulation capacity. Coping.adjustability lowers reappraisal
    threshold. Backward compatible (None defaults).
    Module: src/pathos/engine/generator.py, regulation.py

LEVEL 2 — REPRESENTATION ENGINEERING (local models only):

  Steering Vectors:
    Extracts emotional direction vectors from transformer hidden states
    using contrastive pairs (mean(positive) - mean(negative)). Applied
    via register_forward_hook() during inference. Quadratic intensity
    scaling (intensity² — weak emotions don't steer).
    Extraction: auto via UI toggle, or CLI: python -m pathos.engine.steering_extract --model qwen2.5:3b
    Module: src/pathos/engine/steering.py

  Multi-Layer Steering:
    Different layers get different emotional emphasis:
      Early (0-33%): valence=0.50, arousal=0.30 (lexical/tonal)
      Mid (33-67%): balanced all dimensions (semantic)
      Late (67-100%): dominance=0.35, certainty=0.30 (behavioral)
    Per-layer scaling: early=0.70, mid=1.00, late=0.85.

  TransformersProvider:
    Direct model loading via HuggingFace transformers. Loads HF safetensors
    (not Ollama GGUF — GGUF dequantization has bugs in transformers 4.46-4.48).
    Tries GPU first, falls back to CPU on OOM. Single-device loading only
    (device_map="auto" causes rotary embedding mismatches).
    Exposes steerable_model for hook registration. Loads QLoRA adapters if
    configured. Requires transformers>=4.48.0.
    Compatible architectures: llama, qwen2/2.5, mistral, phi3, starcoder2.
    Module: src/pathos/llm/transformers_provider.py

LEVEL 3 — EMOTIONAL SAMPLING:

  Emotional Sampler:
    Maps emotional state to 6 sampling parameters:
      temperature ← arousal (high → creative, low → measured)
      top_p ← certainty (high → focused, low → explores)
      top_k ← arousal (high → narrow focus, low → broader)
      repetition_penalty ← valence+arousal (rumination → lower)
      presence_penalty ← dominance+valence (open → new topics)
      frequency_penalty ← arousal (high → allows emphasis)
    Intensity gates all: <0.05 → base values unchanged.
    Module: src/pathos/engine/emotional_sampler.py

  Token-Level Logit Bias:
    6 vocabulary categories (positive, negative, high_energy, low_energy,
    uncertainty, assertive) with ~30-37 bilingual words each. Biases
    clamped to [-2.5, +2.5]. Resolved to token IDs via tokenizer.
    Data: src/pathos/sampling_data/emotional_vocabulary.json

LEVEL 4 — ATTENTION MODULATION (local models only):

  Emotional Attention:
    7 attention categories (threat, agent, positive, negative, loss,
    novelty, uncertainty_markers). Broadening/narrowing from Fredrickson:
    high arousal = narrowing (amplified biases), low arousal = broadening
    (attenuated biases). Hidden states scaled at token positions.
    Scale factor: 1.0 + bias*0.10, clamped [0.70, 1.30].
    Data: src/pathos/sampling_data/attention_vocabulary.json
    Module: src/pathos/engine/emotional_attention.py

LEVEL 5 — PREDICTIVE & RECURRENT:

  Emotional World Model:
    3-step causal chain before sending response:
      1. Predict self-impact (empathy→relief, confrontation→discomfort)
      2. Predict user-impact (content analysis, rapport/trust modulation)
      3. Meta-reaction (anticipatory guilt/satisfaction, spiral detection)
    Risk threshold=0.55, alignment threshold=0.50. Max 1 retry shared
    with self-appraisal. OFF in raw/extreme mode.
    Module: src/pathos/engine/world_model.py

  Steering Momentum:
    Temporal inertia: blended = (1-momentum)*current + momentum*history.
    Exponential decay on history (most recent = highest weight). Max 5
    turns of history. Momentum factor from neuroticism [0.10, 0.50]:
    high neuroticism = emotions linger, low = adapts quickly.
    Module: SteeringMomentum in src/pathos/engine/steering.py

  Soft Emotional Prefix:
    Injects 2 synthetic tokens at embedding layer. Composed from earliest-
    layer steering vectors. Complements steering (context vs processing).
    Scale=0.5, symmetry breaking between tokens. Requires steering vectors.
    Module: src/pathos/engine/emotional_prefix.py

  Training Infrastructure (offline, 24GB+ VRAM):
    Dataset generator: 45 seed scenarios, keyword appraisal, personality
    variations. QLoRA script: 4-bit quantization (NF4), LoRA r=16 alpha=32.
    41 conditioning tokens (<V+1>..<V+5>, <A±N>, <D±N>, <C±N>).
    Adapter loading in TransformersProvider with auto token resize.
    Module: src/pathos/training/

6. HOMEOSTASIS AND REGULATION
===============================================================================

6.1 NATURAL DECAY (passive)
  - Each turn: valence/arousal decay 15% toward baseline
  - If intensity < 0.1: emotion is reclassified to neutral
  - Body state: energy drains gradually, tension decays

6.2 BASELINE SHIFT
  - Extreme events (intensity > 0.8) move the baseline
  - Sensitization: more events = faster shift
  - Max drift: 0.5. Recovery: after 5 turns without extreme

6.3 ACTIVE REGULATION
  - 4 strategies with different costs (ego depletion)
  - Breakthrough when capacity < 0.15 (emotion amplified 15%)

6.4 REAPPRAISAL
  - Activates when intensity > 0.6, dominance > 0.35
  - Transforms: fear->alertness, anxiety->contemplation, etc.

6.5 MOOD SYSTEM
  - Evolves based on emotional history (10 snapshots)
  - Congruence bias: 15% bias on new emotions

6.6 EMOTION DYNAMICS ODE
  - dx/dt = -k*(x-att) + noise + perturbation + contagion
  - Inertia by emotion: anger=0.8, surprise=0.1

6.7 EMOTIONAL IMMUNE SYSTEM
  - Detects trauma: intensity > 0.7 AND valence < -0.3
  - Numbing (3+ turns), Dissociation (6+), Compartmentalization (8+)
  - Automatic recovery: protection_strength -0.15/turn without trauma

9. VOICE SYSTEM
===============================================================================

9.1 VOICE MODES
  text_only:   No voice (default)
  voice_out:   TTS — the agent speaks with emotion (V1)
  full_voice:  TTS + ASR — the agent speaks and listens (V1+V2)

9.2 TTS (Voice V1)
  Two backends:

  Kokoro TTS (primary):
    - 82M params, ~350MB VRAM
    - Multi-language: EN works out of the box
    - Other languages (ES, DE, FR, IT, JA, KO, NL, PL, PT, HI) need espeak-ng
    - Voice keys: af_heart, am_adam, ef_dora, em_alex, etc (Kokoro naming)
    - Used for basic/standard emotions
    - Audio: 24kHz, PCM16, mono WAV

  Parler-TTS (optional, complex emotions):
    - ~880M params, ~2.5GB VRAM
    - EN only — text description control
    - Used when emotion is complex or mixed AND language is EN
    - Falls back to Kokoro if language is not EN

  Voice Parameter Generator: emotional_state → voice params
    - speed:           0.85 to 1.25 (arousal/intensity based)
    - pitch_semitones:  -1.5 to +1.5 (scipy resample post-processing)
    - volume:          70% to 120% (modulated by emotion)
    - tremolo:         applied for fear/anxiety states
    - Stage directions: "[speaking with warmth and a smile]" etc.

  Backend selection: automatic
    - Basic emotions → Kokoro
    - Complex/mixed emotions + EN → Parler-TTS
    - Complex/mixed emotions + non-EN → Kokoro fallback

  Dynamic voice catalog: only shows languages that work on the system
    (checks espeak-ng availability at startup)

  Auto-play: audio plays automatically on response

9.3 ASR (Voice V2)
  - Whisper-small (OpenAI, openai-whisper package)
  - Audio input: WAV or WebM (any sample rate, mono/stereo)
  - Automatic resampling to 16kHz
  - Automatic language detection
  - Output: transcribed text + segments with timestamps
  - ~0.5GB additional VRAM

9.4 FRONTEND VOICE COMPONENTS
  VoiceOrb:
    - Web Audio API AnalyserNode, FFT reactive visualization
    - Visual feedback of audio output in real-time

  MicConfigPanel:
    - Device selector for audio input
    - Volume test (visual level meter)
    - Stream sharing across components

  MicInput:
    - Instant recording via pre-acquired stream
    - No delay on record start (stream already acquired)

9.5 HARDWARE (GTX 1660 Super 6GB)
  text_only:   qwen3:4b (2.5GB) = 2.5GB
  voice_out:   + Kokoro (~350MB) = ~2.85GB
  full_voice:  + Whisper (0.5GB) = ~3.35GB
  voice_out with Parler: + Parler-TTS (~2.5GB) = ~5.0GB
  Requirement: espeak-ng for non-English languages

===============================================================================

  1. The system NEVER claims to "feel" in a conscious sense
  2. Full transparency: the user can see all internals
  3. No emotional manipulation of the user
  4. User distress takes PRIORITY over agent's emotional authenticity

  6. SER (vocal emotion detection) is always OPT-IN, never forced
  7. The user can deactivate any advanced system
  8. External signals (webcam, keyboard) are ALWAYS OPT-IN with explicit consent
  9. API consumers must be informed this is functional, not conscious emotion
